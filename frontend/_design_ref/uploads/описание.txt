1. Публичные страницы
/login — LoginPage
UI: split-layout (brand-panel + form), logo «Grid Bench», badges, поля email/password, checkbox «Запомнить», ссылка на reset, ошибка, CTA.
Логика: login() → JWT (setAuthTokens), remember-me (localStorage), onAuthSuccess → getMe, редирект ADMIN→/admin, иначе safeFrom или /tasks.
События: onSubmit, onChange полей, Link → register/reset, auto-redirect если уже auth.
Замечание: после логина цель /tasks, но маршрут /tasks редиректит на / (работает, но путь лишний).

/register — RegisterPage
UI: центрированная форма (имя, email, пароль, инвайт-код опционально), валидация inline, CTA, ссылка на login.
Логика: register() → auto-login; опционально joinGroupByCode() → /student/groups/:id или profile с ошибкой; иначе /.
События: onSubmit, onChange, onBlur пароля, Link.

/reset-password — ResetPasswordPage
UI: иконка, email input, success-state, back link.
Логика: requestPasswordReset(email) — только запрос письма (нет UI смены пароля по токену).
События: onSubmit, Link → login.

2. Студент / общий каталог
/ — TasksPage
UI: TasksHeader, TasksFilters (типы + LanguageFilters), TasksTable, боковая AssignmentSetsPanel.
Логика:

загрузка: getTasks, getTaskTypes, listAccessibleAssignmentSets
фильтры client-side: тип, search, status (solved/todo/attempted), difficulty, patterns (any/all)
langFrom/langTo не участвуют в filteredTasks — только UI-state
«Продолжить обучение»: getNextRecommendation() → navigate adaptive
stats: solved/total
События: click row → /tasks/:id, continue learning, sign out, nav profile/admin, swap langs, apply/reset advanced filters.

Подкомпоненты: TasksHeader, TypeButtons, LanguageFilters, TasksTable, AssignmentSetsPanel.

/tasks/:id — TaskPage + useTaskSolver
UI: TaskHeader, banner teacher-review, TaskLeftPanel, TaskRightPanel, FAB Run (Play), TestPanel, swap-langs (не blocks/algorithm).
Режимы редактора (по task.type):

blocks → BlockEditor (flowchart)
block_reorder / code_assembly / task_build_from_blocks → BlockAssemblyEditor
иначе → CodeEditorBoard (Monaco)
Логика:

load task, drafts localStorage, navigation context (sessionStorage: adaptive | manual + collectionId)
submit: createSubmission, submitBlockReorderSolution, checkFlow, lintSolution
teacher review: location.state.teacherReview → read-only, load submission
adaptive next: getNextRecommendation; manual: getCatalogNavigation
language swap, tab switching (task/examples/flow, tests/result/errors)
События: run code, prev/next task, back to /, tab clicks, language selects, block placement changes.

Подкомпоненты: TaskHeader, TaskLeftPanel, TaskRightPanel, TestPanel.

/student/profile — StudentProfilePage
UI: header (back, logout), sidebar (UserAvatar, stats Card), GithubContributionGraph, tabs: Прогресс / Мои решения / Группы.
Логика: getStudentProfile, getStudentAnalytics (tab progress), panels по табам.
События: tab switch, location.state.tab/error (после register).

Feature panels: StudentProgressPanel, StudentSolutionsPanel, StudentProfileGroupsTab.

/student/groups/:groupId — StudentGroupDetailPage
UI: header, Card преподавателя, StudentGroupCatalogsView.
Логика: getMyGroupWorkspace(groupId).
События: Link back, open tasks from catalogs (внутри feature).

Не дублирует полный TaskEditorPage — узкий legacy-редактор.

4. Админ (AdminLayout sidebar)
Общий shell: sidebar NavLink, email, «← К задачам».

/admin/users — AdminUsersPage
UI: search, role/status selects, checkbox deleted, DataTable, Pagination.

/admin/users/:userId — AdminUserDetailPage

Логика: getAdminUser, setAdminUserBlocked, deleteAdminUser, assignAdminUserRole, removeAdminUserRole.

/admin/teacher-requests — AdminTeacherRequestsPage
UI: status filter, table, approve/reject per row, confirm.

/admin/assignments — AdminAssignmentsPage

/admin/statistics — AdminStatisticsPage

/admin/create-teacher — AdminCreateTeacherPage

Логика: createTeacher().
